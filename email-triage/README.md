# 邮箱 Triage · 打标 + 待办清单

> 每天早上输入一个词，40 秒后：今天 3 件必办、2 件待跟进、13 封推广已过滤。你读的是清单，不是邮件。

这是作者搭的**第一条**工作流，每天真实在用，也是 [Real Agent Use Cases 第 1 期](https://realagentusecases.com/zh/posts/2026-05-13-gmail-ai-agent/)拆解的案例——门槛最低，最适合当你的第一条。

## 这条工作流长什么样

| | |
|---|---|
| **触发** | 手动：每早在 agent 里输入触发词（如 `triage`）；也可挂成每日定时 |
| **步骤** | 拉近 24 小时邮件 → 分四类（Action Required / 本周跟进 / FYI / 可忽略）→ 打标签（可选）→ 生成 P0/P1/P2 待办 |
| **产出** | 一份带时间窗口和原邮件直达链接的当日待办清单 |
| **人工** | 扫一眼清单，把动作抄进你的任务 app，3 分钟排好一天 |

## 设计要点（为什么长这样）

- **先分四类、再排三级，两步走是故意的**。直接让模型从一堆邮件里拉优先级列表，输出会很乱；先宽松分桶、再严格排序，稳定得多。
- **Sources 一节不能省**。每个优先级判断都要有一键回到原邮件的链接——没有这条，你不会信任输出。
- **「不为凑数编造任务」比看起来重要**。没这条规则，邮箱清净的日子它会编一堆低价值任务撑场面。
- **规则区的每一条都是翻车换来的**：老板的 "thanks!" 进过 P0、面试确认漏在 12 封回复深处、"Final notice" 营销标题被误判成紧急……详见[原文的翻车段](https://realagentusecases.com/zh/posts/2026-05-13-gmail-ai-agent/)。
- **改 prompt，别改输出**。每改一次 prompt，明天的 triage 永久变好一点；手动改今天的输出，明天一点都不会变。

## 怎么装

作者的零代码方案（约 20 分钟）：

1. Claude Pro + 设置里连上 Gmail 连接器（只给读权限）
2. 新建一个 Project，把 [`SKILL.md`](./SKILL.md) 的内容贴进 Project 指令
3. 把「我的背景」一节改成你自己的岗位、老板、活跃项目、跳过名单
4. 每天早上输入触发词；头两周每次误判就回去加一条规则

用其他邮箱 / 其他 agent 工具（Codex、OpenCode 等）：换成对应的邮箱连接器和触发方式，prompt 结构不变。技术栈对比（Apps Script / Zapier / n8n / 连接器四条路线的成本和取舍）见[原文](https://realagentusecases.com/zh/posts/2026-05-13-gmail-ai-agent/)。
