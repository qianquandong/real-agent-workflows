# Obsidian · 短视频稿输出（可选）

> 输入端把素材备齐之后，这条负责把素材变成能直接开录的逐字稿。backlog 永远比你拍得快。

这是 [AI 新闻输入](../obsidian-ai-news-input/) 的**可选下游**：只想要一个自动更新的 AI 资讯库，装输入端就够了；想让库里的素材变成内容产出，再加这条。

## 这条工作流长什么样

| | |
|---|---|
| **触发** | 每天定时（作者设在下午） |
| **步骤** | 搜全库选题（查台账防重复）→ 中英各写一版 1 分半逐字稿 → 落盘 + 更新待拍清单 |
| **产出** | `zh-90s.md` + `en-90s.md` + `topic.md`（选题理由、数字出处、CTA） |
| **人工** | 拍摄时挑稿 + 出镜照着念——选题把关移到拍摄端，不卡产能 |

## 怎么装

1. Claude Code 用户：把 [`SKILL.md`](./SKILL.md) 复制到 `~/.claude/skills/obsidian-script-output/SKILL.md`；其他 agent 工具见[根 README 的通用说明](../README.md#怎么用)
2. 替换四个占位符：库路径、素材文件夹、输出文件夹、你的领域
3. 手动跑一次，检查稿子是否符合「出镜照着念」的标准，再挂定时

## 设计要点（为什么长这样）

- **逐字稿标准**：稿里不能出现念不出来的东西——标题、bullet、括号注释都不行，数字写成读法。这不是洁癖，是拍摄时零卡顿的前提。
- **中文不从英文直译**：两版各自按口播语感原生重写。直译的中文稿念出来就是翻译腔，观众三秒划走。
- **台账防重复**：不止防选题重复，还防「连续三天用同一个开头公式」这种隐性重复。
- **人不进生产环节**：人只出现在两个位置——拍摄时挑稿、出镜念稿。其余全自动。
