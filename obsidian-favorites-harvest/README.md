# Obsidian · 收藏收割

> 小红书和抖音里点过收藏就再没打开过的内容，每周自动抓出来、筛掉纯娱乐、转成文字进知识库。把「点过收藏」变成「真的可查」。

## 这条工作流长什么样

| | |
|---|---|
| **触发** | 每周定时（作者设在周三 03:00 低峰） |
| **步骤** | 登录态浏览器抓两个平台的收藏清单 → 台账过滤只留新增 → 标题筛掉纯娱乐 → 视频转写 / 图文逐图转录 → 知识卡入库 |
| **产出** | `YYYY-MM-DD_平台_标题.md` 知识卡：为什么收藏 / 摘要 / 全文转写 / 候选结论 |
| **人工** | 0 分钟（拿不准的内容会列成「待定」等你定） |

作者实测第一次跑：两个平台合计 450+ 条存量收藏，约三分之二是纯娱乐被自动筛掉，教学干货子集逐周滚动消化。

## 设计要点（为什么长这样）

- **绝不代登录**：页面要扫码就跳过该平台、报告里提醒，这是安全红线。
- **只入教学干货**：收藏夹是冲动的，知识库是严肃的。筛选这步不做，库一周就变垃圾场。
- **转完即删媒体**：只留文字稿，硬盘不背几百个视频。
- **台账保证幂等**：`fav-ledger.json` 记录处理过的 ID，跑一百次也不会重复入库；单次最多处理 4 条，量大自动分批到下次。
- **不镜像删除**：从收藏夹移除的内容，库里照留——库是积累，不是同步盘。

## 怎么装

1. 把 [`SKILL.md`](./SKILL.md) 复制到 `~/.claude/skills/obsidian-favorites-harvest/SKILL.md`
2. 替换占位符：库路径、知识卡文件夹、你的小红书收藏页 URL、领域清单
3. 需要 [Claude in Chrome](https://claude.com/chrome) 扩展（要用你真实浏览器的登录态）+ `yt-dlp` + Whisper
4. 先手动跑一次（首次是全量 backfill，会分批），确认知识卡质量再挂定时
