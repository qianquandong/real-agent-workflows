# Obsidian · 收集箱入库

> 白天刷到什么好东西——文章、YouTube 视频、抖音、小红书、一本书——把链接粘进一个页面就完事。晚上它自动抓原文、转文字、变成知识卡进库。

## 这条工作流长什么样

| | |
|---|---|
| **触发** | 手动一句「清收集箱」，或挂成每晚定时 |
| **步骤** | 读收集箱「待处理」区 → 按类型抓文/转写 → 生成知识卡入库 → 链接挪进「已处理」 |
| **产出** | 每条链接一张知识卡：为什么收藏 / 摘要 / 全文转写 / 候选结论 |
| **人工** | 粘链接（顺手加一句 `// 备注` 说明为什么存） |

支持的链接类型：网页文章、YouTube、抖音、B站、小红书图文、书（书只抓简介目录，不编内容）。

## 设计要点（为什么长这样）

- **收集的摩擦必须趋近于零**。粘一个链接是你能坚持一年的动作；「打开笔记软件建卡填模板」不是。收集和整理分离，整理交给机器。
- **`// 备注` 是最值钱的一行**。三个月后你不会记得为什么存这条——粘的时候顺手写半句，知识卡的「为什么收藏」就会回应它。
- **页面本身就是状态机**：待处理 / 已处理 / 失败三个小节，失败记录带 `retry=1/2`，处理进度一眼可见，不需要任何外部数据库。
- **失败两次就停**。第一次失败记为 `retry=1`，下次自动再试；第二次仍失败就记为 `retry=2`，留给人处理。
- **转完即删媒体**：媒体只进本次任务的独立临时目录，清理时也只删本次创建的文件，硬盘不背视频，更不会误伤用户文件。

## 怎么装

1. Claude Code 用户：把 [`SKILL.md`](./SKILL.md) 复制到 `~/.claude/skills/obsidian-inbox-ingest/SKILL.md`；其他 agent 工具见[根 README 的通用说明](../README.md#怎么用)
2. 在库里建收集箱页面（三个小节的模板在 SKILL.md 里）
3. 替换占位符：库路径、收集箱文件、知识卡文件夹、领域清单
4. 视频转写需要 `yt-dlp` + Whisper；小红书图文需要能控制已登录浏览器的扩展
5. 粘两三个链接手动跑一次，确认知识卡质量再挂定时

## 和其他工作流的关系

- 和[首页驾驶舱](../obsidian-morning-cockpit/)互补：早报勾选收「推给你的」，收集箱收「你自己刷到的」——两个入口，同一个知识库。
- 和[收藏收割](../obsidian-favorites-harvest/)的区别：收藏收割是每周批量清平台收藏夹，这条是随手一条条投喂。转写和知识卡逻辑一致。
